# Quickstart

Get started with ZOODOM and have your development environment running in under 5 minutes.

## Prerequisites

Before you begin, ensure you have:

- Node.js version 22.12.0 or higher
- npm, yarn, or pnpm package manager
- A code editor (VS Code recommended)

<Info>
  Check your Node.js version with `node --version`. ZOODOM requires Node.js >= 22.12.0 as specified in package.json.
</Info>

## Quick Start

<Steps>
  <Step title="Clone the repository">
    Clone the ZOODOM repository to your local machine:

    ```bash
    git clone https://github.com/Iceheop/zoodom.git
    cd zoodom
    ```
  </Step>

  <Step title="Install dependencies">
    Install the project dependencies using your preferred package manager:

    <CodeGroup>
    ```bash npm
    npm install
    ```

    ```bash yarn
    yarn install
    ```

    ```bash pnpm
    pnpm install
    ```
    </CodeGroup>

    This will install Astro 6.0.4 and all required dependencies.
  </Step>

  <Step title="Start the development server">
    Run the development server:

    <CodeGroup>
    ```bash npm
    npm run dev
    ```

    ```bash yarn
    yarn dev
    ```

    ```bash pnpm
    pnpm dev
    ```
    </CodeGroup>

    The site will be available at `http://localhost:4321`.
  </Step>

  <Step title="Explore the site">
    Open your browser and navigate to `http://localhost:4321` to see the ZOODOM homepage.

    You should see:
    - The responsive navigation bar with logo
    - The hero section with "Bienvenido a nuestro zoológico"
    - The Bootstrap-styled interface
  </Step>
</Steps>

## Development Commands

Here are all available npm scripts from package.json:

| Command | Description |
|---------|-------------|
| `npm run dev` | Start the Astro development server at http://localhost:4321 |
| `npm run build` | Build the production site to the `dist/` directory |
| `npm run preview` | Preview the production build locally |
| `npm run astro` | Run Astro CLI commands |

<Tip>
  Use `npm run dev` during development for hot module reloading. Changes to Astro components, CSS, and assets will reflect immediately.
</Tip>

## Build for Production

When you're ready to deploy, build the production site:

<CodeGroup>
```bash npm
npm run build
```

```bash yarn
yarn build
```

```bash pnpm
pnpm build
```
</CodeGroup>

This creates an optimized production build in the `dist/` directory.

## Troubleshooting

<Accordion title="Port 4321 is already in use">
  If port 4321 is already in use, Astro will automatically try the next available port (4322, 4323, etc.). Check the terminal output for the actual URL.
</Accordion>

<Accordion title="Module not found errors">
  Try deleting `node_modules/` and the lockfile, then reinstalling:
  
  ```bash
  rm -rf node_modules package-lock.json
  npm install
  ```
</Accordion>

<Accordion title="Node version mismatch">
  Ensure you're running Node.js >= 22.12.0:
  
  ```bash
  node --version
  ```
  
  If your version is lower, update Node.js using nvm or your preferred method.
</Accordion>
