# Layout System

ZOODOM uses a centralized `Layout.astro` component to provide consistent structure, Bootstrap dependencies, and styling across all pages.

## Layout Component

The Layout component serves as a wrapper for all page content:

```astro
---
import Head from "../components/Head.astro";
---

<!doctype html>
<html lang="es">
  <head>
    <Head />
    <link
      href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.8/dist/css/bootstrap.min.css"
      rel="stylesheet"
      integrity="sha384-sRIl4kxILFvY47J16cr9ZwB07vP4J8+LH7qKQnuqkuIAvNWLzeN8tE5YBujZqJLB"
      crossorigin="anonymous"
    />
  </head>
  <body>
    <slot />
    <script
      src="https://cdn.jsdelivr.net/npm/@popperjs/core@2.11.8/dist/umd/popper.min.js"
      integrity="sha384-I7E8VVD/ismYTF4hNIPjVp/Zjvgyol6VFvRkX/vR+Vc4jQkC+hVqc2pM8ODewa9r"
      crossorigin="anonymous"></script>
    <script
      src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.8/dist/js/bootstrap.min.js"
      integrity="sha384-G/EV+4j2dNv+tEPo3++6LCgdCROaejBqfUeNjuKAiuXbjrxilcCdDz6ZAVfHWe1Y"
      crossorigin="anonymous"></script>
    <script
      src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.8/dist/js/bootstrap.bundle.min.js"

      crossorigin="anonymous"></script>
  </body>
</html>
```

## Astro Slot System

The `<slot />` tag is where page content is injected:

```astro
<body>
  <slot />
  <!-- Scripts load after content -->
</body>
```

<Info>
  The slot acts as a placeholder that gets replaced with the content from child pages that use this layout.
</Info>

## Using the Layout

Pages import and wrap their content with the Layout component:

```astro
---
import Layout from '../layouts/Layout.astro';
import logo from'../assets/logo.svg';
import HeroImg from '../assets/hero.png';
---

<Layout>
  <main>
    <!-- Page content goes here -->
  </main>
</Layout>
```

## Bootstrap Dependencies

The layout includes Bootstrap 5.3.8 from CDN:

<Tabs>
  <Tab title="CSS">
    ```html
    <link
      href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.8/dist/css/bootstrap.min.css"
      rel="stylesheet"
      integrity="sha384-sRIl4kxILFvY47J16cr9ZwB07vP4J8+LH7qKQnuqkuIAvNWLzeN8tE5YBujZqJLB"
      crossorigin="anonymous"
    />
    ```
    Loaded in `<head>` for immediate style application.
  </Tab>
  <Tab title="JavaScript">
    ```html
    <!-- Popper.js for tooltips/popovers -->
    <script src="https://cdn.jsdelivr.net/npm/@popperjs/core@2.11.8/dist/umd/popper.min.js"></script>
    
    <!-- Bootstrap JS -->
    <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.8/dist/js/bootstrap.min.js"></script>
    
    <!-- Bootstrap Bundle (includes Popper) -->
    <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.8/dist/js/bootstrap.bundle.min.js"></script>
    ```
    Loaded at the end of `<body>` for optimal page load performance.
  </Tab>
</Tabs>

<Note>
  Bootstrap scripts are loaded after the slot content to ensure DOM elements are available before JavaScript initialization.
</Note>

## Head Component Integration

The layout imports a reusable `Head` component:

```astro
---
import Head from "../components/Head.astro";
---

<!doctype html>
<html lang="es">
  <head>
    <Head />
    <!-- Bootstrap CSS -->
  </head>
```

<Tip>
  The `Head.astro` component likely contains meta tags, title, and other common head elements, promoting DRY principles.
</Tip>

## Benefits of This Layout Pattern

| Benefit | Description |
|---------|-------------|
| **Consistency** | All pages share the same HTML structure and dependencies |
| **Maintainability** | Bootstrap version updates happen in one place |
| **Performance** | Scripts load after content for better perceived performance |
| **Internationalization** | `lang="es"` attribute set globally for Spanish content |
| **Security** | Subresource Integrity (SRI) hashes verify CDN resources |

## Script Loading Strategy

The layout loads multiple Bootstrap scripts:

1. **Popper.js** - Required for positioning tooltips and popovers
2. **Bootstrap.min.js** - Core Bootstrap JavaScript
3. **Bootstrap.bundle.min.js** - Bootstrap + Popper combined

<Info>
  While `bootstrap.bundle.min.js` includes Popper, the separate scripts provide flexibility for components that need specific versions.
</Info>

## Language Setting

```html
<html lang="es">
```

The layout sets Spanish as the document language, which:
- Helps screen readers pronounce content correctly
- Assists search engines with language detection
- Improves accessibility and SEO

<Tip>
  When creating multilingual sites, you can pass the `lang` attribute as a prop to the Layout component.
</Tip>
